# 心情模块开发流程文档

## 1. 目标与范围

### 本次目标

- 为桌宠增加可持续的“心情模块”，并将其保存到本地。
- 在自动模式下，用户右键点击宠物即可打开互动面板。
- 用户和宠物互动后，宠物会立刻给出动作、气泡和心情变化反馈。
- 宠物重启后仍能恢复上次的心情状态。

### 第一版范围

- 单一心情值模型：`0-100`
- 心情档位：`低落 / 平静 / 开心 / 兴奋`
- 互动动作：`摸摸头 / 夸夸它 / 陪它玩 / 喂点心 / 让它休息 / 聊聊天`
- 本地持久化：`config/mood_state.json`
- 自动动作权重联动
- 聊天语气联动

### 第一版暂不做

- 多维养成系统（饥饿、清洁、社交、金币等）
- UI 自动化测试
- 分身模式下的独立心情副本

## 2. 用户场景

### 场景 A：自动模式快速互动

1. 用户将宠物切到自动模式。
2. 用户右键点击宠物。
3. 宠物弹出互动面板，显示当前心情、最近互动、今日陪伴提示。
4. 用户点击任一互动动作。
5. 宠物立即播放对应动作并弹出反馈气泡。
6. 后续自动动作更符合当前心情。

### 场景 B：重启恢复

1. 用户与宠物互动若干次。
2. 关闭并重新打开应用。
3. 宠物恢复上次心情值、最近互动和今日计数。

### 场景 C：聊天更像“活的宠物”

1. 用户通过互动面板点击“聊聊天”。
2. 宠物进入倾听状态并打开现有聊天框。
3. AI 回复会自然带上当前心情语气。

## 3. 交互方案

### 互动面板结构

- 顶部：宠物名 + 当前心情标签
- 中部：心情描述、心情值进度条、最近互动
- 辅助信息：今日陪伴提示
- 底部：六个快捷互动按钮

### 交互反馈

- 点击互动后立即更新面板数值
- 宠物气泡显示反馈文案
- 宠物短暂进入建议动作
- 自动模式短时间内不抢走互动动作

## 4. 状态模型

### 核心字段

- `pet_id`
- `mood_score`
- `updated_at`
- `last_interaction`
- `last_interaction_at`
- `interaction_counts`
- `recent_actions`
- `daily_date`
- `daily_interactions`

### 状态规则

- 默认心情值：`58`
- 随时间向“平静区间”缓慢回落
- 重复执行同一互动时收益递减
- 每次互动后立即保存到本地

## 5. 数据存储设计

### 文件位置

- `config/mood_state.json`

### 存储结构

- 顶层按 `pet_id` 分组
- 每只宠物独立保存自己的心情快照
- 文件损坏或字段缺失时，自动回退到默认状态

## 6. 实现拆分

### 产品经理

- 定义第一版最小闭环
- 明确互动动作、反馈方式、验收标准

### 前端开发

- 新增 `peko/ui/mood_dialog.py`
- 实现右键互动面板 UI
- 保持和现有暖色圆角面板风格一致

### 后端开发

- 新增 `peko/core/mood.py`
- 实现心情状态、规则、持久化、自动动作偏好
- 将心情逻辑接入 `pet.py`、`auto.py`、`chat.py`

### 测试工程师

- 为纯逻辑补单测
- 输出人工验收清单
- 验证右键交互、持久化和模式切换不回归

## 7. 开发步骤

1. 新增心情核心模块，先跑通默认状态、互动规则和 JSON 持久化。
2. 新增互动面板 UI，并提供当前心情展示与按钮点击事件。
3. 在 `pet.py` 中接入右键事件、互动状态锁、气泡刷新和面板刷新。
4. 在 `auto.py` 中引入基于心情的动作权重扩展。
5. 在 `chat.py` 中注入心情上下文，使回复更贴合宠物状态。
6. 补充单测并执行编译检查。
7. 进行人工验收。

## 8. 验收标准

- 自动模式下右键宠物可以打开互动面板。
- 面板能展示当前心情、最近互动和今日提示。
- 六种互动按钮都能触发即时反馈。
- 心情值会被保存到本地，重启后仍然保留。
- 自动模式下的动作表现会随心情变化。
- 聊天时会带有当前心情语气。
- 不影响现有拖拽、三击 fight、托盘模式切换、跟随鼠标和操控模式。

## 9. 风险与约束

- `pet.py` 已较大，新增逻辑必须尽量下沉到 `core/mood.py`。
- 右键交互不能破坏左键拖拽和三击 fight。
- 自动动作与手动互动之间需要短暂锁定，避免状态被立刻覆盖。
- 分身模式下共享主宠心情，不单独持久化。
